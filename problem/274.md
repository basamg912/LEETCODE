h-index
h개 페이퍼가 h번 이상 인용되었을 경우. 가장 큰 h return
`[3,0,6,1,5]` -> 3개 페이퍼가 3 이상 인용되었으니 hindex=3, 2개 페이퍼가 2번 이상 인용되었으니 2도 맞지만, 젤 큰 3 return, 4개 페이퍼가 4번 이상 인용된 경우 X 이니 4는 X

Approach
- citation > rest  상황 처리를 생각해내는게 어려웠다. 테스트케이스에서는 `[100]` 에서 output = 1이 나와야하는데, 처음 코드에서는 `citation=100 > rest=1` 에 대한 처리를 안해두어서 0 이 출력되었음.
- 위 예외 처리가 필요한 가장 이해하기 쉬운 케이스는 `[5,5,5]` 에 대한 처리. citation 은 3개 논문 모두 5 이상을 만족하지만, 논문 갯수가 3개 뿐이니 h-index 는 3 이 되어야한다. 원래 코드에서는 위 코드도 처리하지못해서 0 이 출력되었다.
- h-index 에 citation 말고도 rest 가 기여할 수 있도록 else 문을 추가로 작성했다.
