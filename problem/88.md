```cpp
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
        vector<int> temp(m+n);
        int p = m-1;
        int q = n-1;
        int idx = m+n-1;
        while (p>=0 && q>=0){
            if (nums1[p] >= nums2[q]){
                temp[idx--] = nums1[p--];
            }
            else{
                temp[idx--] = nums2[q--];
            }
        }
        while (p >= 0){
            temp[idx--] = nums1[p--];
        }
        while (q >= 0){
            temp[idx--] = nums2[q--];
        }
        nums1 = temp;
    }
};
```
Approach 1
- `vector<int> temp(m+n)` 생성하고, nums1, nums2 값 비교해서 temp 에 넣고, `nums1=temp`
  - 근데 위 방식은 새로운 temp 벡터를 선언하게 되면서 공간복잡도가 O(m+n) 증가하게 된다.
  - 실수할 일 없이 직관적인 코드를 먼저 작성해봤는데, 먼저 reference 로 받는 nums1 의 경우 temp 값을 복사해서 사용하게된다. 처음에는 `vector<int> *nums1` 포인터로 받아서 `nums1=&temp` 를 하면 좋겠다 싶었는데, temp 는 로컬 변수라서 함수 종료되면 스택 영역 (temp 변수 자체가 위치) 과 힙 영역 (vector 위치) 에 메모리영역이 destructor 가 호출되면서 free 된다. 그래서 위처럼 포인터 주소를 바꿔주는건 undefined behavior. 포인터륿 매개변수로 받아도 마찬가지로 `*nums1 = temp` 로 역참조해서 temp 를 복사하는 방식을 해야함

Approach 2
- nums1 의 뒤에서부터 채우는 방식. 두 배열이 정렬되어있는 상태니깐 nums1 의 인덱스 p 에 대해서는 상관안해도 되고 마지막에 q 가 남았을때만 while 문 돌리는 것 인상적이였다.
